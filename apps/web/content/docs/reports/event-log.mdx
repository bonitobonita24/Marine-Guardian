The **Event Log** report is the detailed record — a line-by-line export of the
events captured over a chosen date range. Use it for audits, incident reviews,
or handing a complete activity trail to an oversight body.

## Who uses it

- **Operators, coordinators, and viewers** generate the event log of data they
  can already see.
- **Site admins / super admins** manage the export jobs.

## Key actions

1. Choose the **Event Log / Detailed** report type (stored internally as the
   `detailed` report type).
2. Set the date range and any filters that scope which events are included.
3. Pick a paper size — **A4** (default), **Letter**, or **Legal**.
4. Submit and download from the Exports history once the job is *ready*.

Every generated export is recorded in the tenant's audit trail, so the act of
producing an event log is itself accountable.
